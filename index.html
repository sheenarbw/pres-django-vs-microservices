What are Microservices

besides fashionable?

Let's imagine...Webflix

You have an application that handles
  • distributing large volumes of videos
  • authentication, authorization
  • Managing customer information
  • communicating with customer
  • keeping track of payments and subscriptions
  • keeping track of who has seen which video
  • making recommendations
  • ...and a whole lot more
  • Each service is its own application
  • Can be different languages
  • Can have different databases
  • communication can be handled in different ways
  • updated independently
  • scaled independently

Real life microservices

https://www.appcentrica.com/the-rise-of-microservices/

Why do this to yourself?

  • Lots of independent teams
  • Need independent and rapid scaling of compute resources
  • Rapid deployment and rollback of small changes
  • helps to enforce loose coupling
  • self healing
  • make you 10% cooler

(some) Pitfalls

  • NEED CI/CD to be done well
  • unit testing gets tricky. Mock all the things
  • integration testing becomes tricky
  • saturation testing becomes necessary
  • networks aren't perfect: messages get dropped, delayed, replayed
  • Observability is hard
  • more distributed and complex system => more time spent debugging production system (== technical debt)
  • Hard to know effect of simultaneous faults
